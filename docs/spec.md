# BT-SYS 仕様書

## 概要

仮想通貨自動トレードシステム。複数ユーザーが各自のボットを管理し、トレード成績の確認とバックテストによる戦略検証を行うWebダッシュボード。

---

## 機能一覧

### 1. 認証・ユーザー管理

- メールアドレス/パスワードによるアカウント登録・ログイン
- ユーザーごとに独立したデータ（ボット・資産・成績）を管理
- ロール区別なし（全ユーザーが同等の権限）
- 取引所APIキーをユーザーごとに安全に管理（暗号化保存）

### 2. 取引所連携

- 初期対応：**bitFlyer**（現物・先物）
- 将来対応：Binance、Bybit 等への拡張を想定した抽象化設計
- 接続テスト機能（APIキー登録時の疎通確認）

### 3. ボット管理

- ボットの作成・編集・削除・起動・停止
- 設定項目：
  - 取引所
  - 通貨ペア
  - トレード種別（現物 / 先物）
  - 戦略の選択
  - 資金枠（予算上限）
  - リスク管理条件（損失が X% を超えたら自動停止 など）
- 複数ボットの同時稼働対応
- ボットのステータス表示（稼働中 / 停止中 / エラー）

### 4. 戦略管理・バックテスト

- 複数の戦略をプリセットとして用意（例：MACD、RSI、グリッド 等）
- 戦略ごとにパラメータを設定可能
- バックテスト機能：
  - 過去データを使った戦略の検証
  - データソースは取引所ごとに異なる（取引所API or 外部データ）
  - 複数戦略の結果を比較・可視化（グラフ表示）

### 5. トレード成績・ダッシュボード

- ボットごとの損益・勝率・取引回数の表示
- 成績の集計（日次・週次・月次・ボット別）※集計単位は後日確定
- 直近の取引履歴一覧
- 資産推移グラフ

### 6. 通知・アラート

- 対応チャンネル：**Discord**、**LINE**
- 通知イベント：
  - 注文約定
  - ボットエラー・停止
  - リスク条件トリガー（自動停止時）
- ユーザーごとに通知設定を管理

## 推奨技術スタック

### バックエンド

- **Python + FastAPI**
  - 仮想通貨ライブラリ（ccxt）が充実
  - 非同期処理でボットの並列稼働に対応しやすい
  - バックテストライブラリ（backtesting.py / vectorbt）が豊富

### フロントエンド

- **Next.js (React) + TypeScript**
  - レスポンシブ対応が容易
  - チャート：TradingView Lightweight Charts または Recharts

### データベース

- **DynamoDB**（ボット状態・取引履歴の永続化、オンデマンドモード）
- Redis は不使用（DynamoDBのステータス管理で代替）

### インフラ（推奨構成）

- **Cloudflare Pages**：フロントエンド（無料・帯域無制限）
- **Lambda + EventBridge Scheduler**：ボット実行（cron、1〜5分間隔）
- **Lambda + API Gateway**：ダッシュボード向けAPI
- **DynamoDB**：ボット状態・取引履歴（オンデマンドモード）
- **Cognito**：認証（5名なら無料）
- 月額目安：$0〜$5（無料枠をうまく使えばほぼ無料）
- 詳細比較・コスト試算：[infra-comparison.md](./infra-comparison.md)

### 取引所ライブラリ

- **ccxt**（bitFlyer含む多数の取引所に対応、マルチ取引所拡張に最適）

## MVP スコープ（フェーズ1）

最初にリリースする最小限の機能：

1. ユーザー認証（登録・ログイン）
2. bitFlyer APIキー登録
3. ボットの作成・起動・停止（1戦略）
4. リアルタイムのボットステータス表示
5. 基本的な成績表示（損益・取引履歴）
6. 簡易バックテスト（1戦略・1通貨ペア）
