# インフラ構成 比較ドキュメント

対象規模：ユーザー数 最大5名、ボット数 数十程度

---

## プラットフォーム比較

| サービス | 月額目安 | 特徴 | 向き/不向き |
|----------|----------|------|-------------|
| **Railway** | $5〜$20 | シンプルな料金体系、PostgreSQL込み | ◎ 小規模・低コスト優先 |
| **Render** | $7〜$25 | 無料枠あり、スリープ機能で節約可 | ○ ダッシュボード向き（ボットはスリープ不可） |
| **Fly.io** | $3〜$15 | 小さいVMを複数立てやすい | ○ ボット分離したい場合 |
| **AWS (ECS+RDS)** | $50〜$150 | 高可用性・柔軟だが設定複雑 | △ 5名規模ではオーバースペック |
| **GCP (Cloud Run+CloudSQL)** | $30〜$80 | サーバーレスでスケール | △ 常時起動ボットとの相性がやや悪い |
| **VPS (Vultr/Hetzner)** | $6〜$12 | 固定費で予測しやすい、自由度高い | ○ 運用コスト最小化したい場合 |

**推奨：Railway**（PostgreSQL・Redis込みで月$10〜$20に収まる見込み）

## Redis（キャッシュ・状態管理）の要否

5名・数十ボット規模では **Redis不要** と判断。

| 用途 | Redisなしの代替 |
|------|----------------|
| ボット状態管理 | PostgreSQLのステータスカラムで管理 |
| リアルタイム通知 | WebSocket + DB polling（数秒間隔）で十分 |
| セッション管理 | JWTトークンで代替 |

将来的にユーザー・ボット数が増えた場合に追加を検討。

## コスト削減の設計方針

1. **フロントエンドとバックエンドを分離**
   - フロントはVercel（無料枠）、バックエンドのみ有料サーバーへ

2. **バックテストはオンデマンド実行**
   - 常時起動不要。リクエスト時のみ実行してリソース節約

3. **取引所データはAPIキャッシュ**
   - 同じデータの重複取得を避け、APIレート制限とコストを削減

4. **ログ・監視は無料枠で**
   - Grafana Cloud（無料枠）または Sentry（無料枠）を活用

5. **Redisは当面不使用**
   - PostgreSQLで代替し、必要になった時点で追加
